# ADR-0005 テンプレート文面とトップページ導線の改善方針

- Status: Proposed
- Date: 2026-03-15

## 背景

Phase 8 の最後の項目は「テンプレート文面とトップページ導線を洗練する」ことにある。現状は generator と検証を優先したため、文面や入口導線は最小構成になっている。

ここで重要なのは、正本を YAML に置く構造を崩さずに、入力テンプレートと公開トップページの両方を少ない変更で改善すること。

## 改善対象

- `templates/fact.yaml`
- `src/README.md` の生成ロジック
- 必要なら `src/404.md` と genre/tag index の補助文

## 提案

最初の改善は schema を増やさず、既存フィールドの見せ方だけを改善する。

## テンプレート文面の方針

`templates/fact.yaml` には次のガイドを加える。

- `summary`: 80〜120 文字目安
- `claim`: 事実主張を 1 文で書く
- `explanation`: 「なぜ意外か」を先に書く
- `sources`: 一次情報を先頭に置く
- `aliases`: 検索されそうな言い換えを入れる

追加はコメント中心にして、スキーマの必須項目は増やさない。

## トップページ導線の方針

トップページは次の順を推奨する。

1. サイトの説明
2. 最近更新 3〜5 件
3. ジャンルから探す
4. タグから探す
5. このサイトの見方

補助コピー例:

- 「根拠付きの“へえー”だけを集める」
- 「公開ページは mdBook、正本は YAML」
- 「気になったらソースをたどれる」

## 推奨理由

- 現在の generator に小さな差分で入れられる
- schema を急いで拡張しなくてよい
- 編集者向けと読者向けの両方の分かりやすさを上げられる

## 将来の拡張余地

- `importance` を使ったピックアップ表示
- 「今日のおすすめ」や「ランダムへえー」
- ジャンル説明文やタグ説明文の taxonomy 追加

## 保留事項

- `editorial` にトップ表示向けフラグを足すか
- テンプレートのコメント量をどこまで増やすか
- 404 ページや About ページを別テンプレートで持つか
